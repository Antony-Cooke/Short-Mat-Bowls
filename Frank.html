Getting Set Up


  1. Download and Install Visual Studio - this is the Web Page Editor
  2. Download and Install GIT - This software allows you to communicate with GITHUB
  3. Register an Account with GitHub
    4.

First Time Only

  1. Go to the Bowls Website home on GitHub https://github.com/Antony-Cooke/Bowls
  2. Click on the "Clone or Download" Button and copy the URL of the Website
  3. Make a folder to store the WebSite on your computer e.g. C:\Web\
  4. Open GIT Bash
  5. Type cd c:\Web then press <ENTER>
  6. Type git clone followed by the URL you copied, above (right-click and select "paste")
  7. Check that the Bowls Website has been cloned into c:\Web\Bowls
    8.

Making Weekly Updates

    If more than one person is making edits to the web site, you need to synchronise with github first.
    • Open Git Bash
      • Type cd c:\Web\Bowls then press <ENTER>
      • Type git pull then press <ENTER>
        • This checks what updates are in GitHub but are not on your computer and merges them onto your machine - that way one person's change does not remove another's

  1. Open Visual Studio
  2. If it isn't already Open, Open the Folder C:\Web\Bowls - Click on File, then Open Folder
  3. All the Available Pages will be listed in the left-hand pane - Single click any one of them to view the content in the central pane

  4. Update Weekly Results
    1. Click on Fixtures in left hand pane
    2. Scroll Down to the Week you want to edit
    3. Put the scores achieved by each of the teams between <td> and </td> on the line below the Home Team name and on the line above the Away Team name
    4. Click on File, then Save

  5. Update League Table
    1. Click on Table in left hand pane
    2. Each line bounded by <tr> and </tr> represents 1 team.
    3. Replace the 9 lines with the 9 lines in Column I on the spreadsheet
    4. When everything looks OK, Click on File, then Save

  6. You can test out the way it looks by double clicking the web page in File Explorer
  7. Once you are happy with how it all looks, you need to publish your pages to GitHub

  8. Open Git Bash
    • Type cd c:\Web\Bowls then press <ENTER>
    • Type git status then press <ENTER>
    • Each Page that you have changed should be listed
    • Type git add . then press <ENTER>
    • The fullstop is important - it means ALL changes
    • Type git commit -m "Description" then press <ENTER>
    • "Description" is something meaningful to describe your changes e.g. "Week 3 Results etc." - the quotes are important
    • Type git push origin master then press <ENTER>
    • This pushes your changes up to GitHub

    I have set up GitHub so that all edits are automatically published Git Pages.
